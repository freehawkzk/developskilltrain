## 你应该知道的概念

### 1. 报错不可怕

Python的报错信息是**帮你的**，不是惩罚你的。它告诉你：
- **什么类型的错误**（最后一行的错误类型）
- **在哪里出错**（traceback中的文件名和行号）
- **为什么出错**（错误消息中的具体信息）

### 2. 常见报错类型

| 报错类型 | 含义 | 常见原因 |
|----------|------|----------|
| `IndexError` | 索引超出范围 | 访问列表/数组中不存在的位置 |
| `KeyError` | 字典键不存在 | 拼写错误或键不存在 |
| `TypeError` | 类型不对 | 字符串和数字直接相加等 |
| `ValueError` | 值不对 | 把非数字字符串转为int |
| `RuntimeError` | 运行时错误 | PyTorch中通常是维度不匹配 |
| `ShapeError` | 形状不匹配 | 张量维度对不上 |

### 3. Git是什么？

Git是一个**版本管理工具**，你可以把它想象成游戏的"存档"功能：
- `git commit` = 存档（保存当前进度）
- `git log` = 存档列表（查看所有存档）
- `git checkout` = 读档（回到某个存档点）

### 4. 好的编程习惯

- 变量名要有意义：`sample_rate` 而不是 `sr`
- 注释解释"为什么"而不是"做了什么"
- 用工具（black）自动格式化代码
