> 请在上课前阅读本文档。

## 你应该知道的概念

### CI相关任务

| 任务 | 输入 | 输出 | 与CI的关系 |
|------|------|------|-----------|
| VAD(语音活动检测) | 音频帧 | 语音/非语音 | ACE策略中"选通道"的前提 |
| 可懂度分类 | 带噪语音 | 可懂/不可懂 | 评价语音增强/编码策略效果 |
| 环境声分类 | 音频 | 环境类型 | 帮助CI用户感知周围环境 |
| 通道选择 | 音频特征 | 刺激/不刺激 | 替代ACE固定规则 |

### 模型调优技巧

| 技巧 | 作用 |
|------|------|
| LR Range Test | 从1e-6逐渐增大到1，观察loss变化，找最佳学习率范围 |
| 混合精度训练(FP16) | 减少GPU内存占用，加速约2倍 |
| 梯度累积 | 小batch多次累积梯度，等效大batch |
| 早停(Early Stopping) | 验证集loss不再下降时停止训练 |

## 核心概念：VAD与ACE策略的连接

```
输入音频 → VAD → 有语音？
                ├── 是 → 执行ACE选通道 → 电刺激
                └── 否 → 不刺激（省电、减少干扰）
```

传统ACE用能量阈值做VAD——安静环境OK，噪声环境容易误判。
DL的VAD学习了更鲁棒的特征，在噪声中也能准确判断。

## 思考题

1. 为什么在噪声环境中，简单的能量阈值VAD会失效？
2. "可懂度"是主观感受，怎么用客观方法来定义和标注"可懂/不可懂"？
3. 混合精度训练为什么能加速？FP16和FP32有什么区别？
