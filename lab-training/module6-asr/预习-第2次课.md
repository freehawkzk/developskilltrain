## 前置要求

### 1. 第1次课内容掌握

确保你理解：
- ASR的基本流程和WER/CER指标
- CTC解决对齐问题的直觉
- Whisper的使用方法

### 2. 环境确认（约10分钟）

```bash
# 确认Whisper已安装
python -c "import whisper; print('OK')"

# 确认能加载模型
python -c "import whisper; m = whisper.load_model('tiny'); print('模型加载OK')"
```

如果模型下载慢，可使用 `faster-whisper` 替代。

### 3. 前续模块输出准备（约10分钟）

确保以下模块的结果可用：

- **模块5**：DeepFilterNet增强后的音频文件（`enhanced/` 目录）
- **模块4**：ACE策略代码可运行，**GET声码器**（`get_voc.py`）可运行
- **测试音频**：至少3段不同SNR的带噪中文语音

### 4. GET声码器预习（约15分钟）

打开 `module4-deepace/ACE/get_voc.py`，快速浏览：

- `get_voc()` 函数的输入参数（电极图 `electrodogram` 和 MAP `map_p`）
- 载波类型：GET（正弦）vs GEN（噪声）
- 高斯包络叠加的过程
- 去加重滤波的作用

> **关键理解**：GET声码器将ACE电极图还原成可听音频。没有这一步，电极图无法送入ASR识别。

### 5. 复习前续模块（约10分钟）

快速回顾：
- 模块4的ACE策略输出格式（`electrodogram` 和 `map_p`）
- 模块4的GET声码器使用方式

## 预习思考题

1. 如果一段语音经过ACE编码后只剩22个通道的包络信息，Whisper还能识别出正确的文本吗？为什么？

2. 设计一个实验来评估"语音增强对ASR识别率的影响"。你会控制哪些变量？用什么指标？

3. 完整Pipeline中，每个模块的误差会如何传播？增强效果不好 → ACE编码受影响 → ASR识别率下降，这个链条中哪个环节最关键？

4. 为什么说ASR准确率不能完全代表CI用户的听懂程度？你能举出一个ASR识别正确但CI用户可能听不懂的例子吗？

5. 如果你要改进这个Pipeline，你会优先改进哪个模块？为什么？

1. **Whisper实战**：加载模型、识别中文语音、查看分段结果
